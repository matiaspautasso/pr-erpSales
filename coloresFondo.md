estructura coherente, pensada para interfaces con tablas, formularios y dashboards.

Base general (fondo y layout)
El fondo principal debería ser un azul muy claro, casi neutro, para no competir con el contenido.

Fondo principal: #EAF4FF
Fondo secundario (cards, paneles): #FFFFFF
Fondo hover / filas activas: #DCEEFF
Bordes suaves: #D0D7E2

Esto genera contraste sin usar grises duros, mantiene coherencia con el azul y funciona bien en pantallas largas.

Tipografía (clave en sistemas de gestión)
El mayor error en ERP suele ser el contraste bajo o mala jerarquía.

Texto principal: #1F2937 (gris oscuro, no negro puro)
Texto secundario: #6B7280
Texto deshabilitado: #9CA3AF
Texto sobre fondo oscuro: #FFFFFF

Esto evita fatiga visual y mejora lectura en tablas extensas.

Colores primarios (acciones y navegación)
El azul debe ser el eje de interacción.

Color primario: #2563EB
Hover primario: #1D4ED8
Activo: #1E40AF

Se usa para botones principales, links, foco de inputs, selección.

Títulos y jerarquía
Los títulos deben diferenciarse sin romper la paleta.

Títulos principales: #1E3A8A
Subtítulos: #1F2937
Labels de campos: #374151

El título no debería ser el mismo azul que los botones, para no confundir acción con estructura.

Estados del sistema (crítico para tu ERP)
Esto es clave para ventas, stock, compras.

Éxito (ej: venta confirmada):
#16A34A (fondo suave: #DCFCE7)

Error (ej: stock insuficiente):
#DC2626 (fondo suave: #FEE2E2)

Advertencia (ej: stock bajo):
#F59E0B (fondo suave: #FEF3C7)

Info (ej: datos informativos):
#0284C7 (fondo suave: #E0F2FE)

Esto tiene que ser consistente en toda la app (no improvisar colores por módulo).

Tablas (muy importante en tu caso)
Porque tu sistema es intensivo en datos.

Fila normal: #FFFFFF
Fila alterna: #F9FAFB
Hover fila: #E0ECFF
Texto tabla: #111827
Header tabla: #F1F5F9

Esto mejora escaneo visual y reduce errores al operar.

Evitar inputs con fondo gris oscuro o bajo contraste.

Recomendación de criterio (esto es lo importante)
No mezclar demasiados azules distintos. Usar 1 primario + variaciones (hover, active).
El fondo debe ser lo más neutro posible dentro del azul claro.
Los colores fuertes (verde, rojo, amarillo) deben reservarse exclusivamente para estados.
Las tablas deben priorizar legibilidad sobre diseño
